The parameter shape
For a Grid layout, the user-facing settings are:

Display type: grid or flex
Children count: integer (informational; affects HTML output)
Rows: count + size (number + unit: fr, px, %, auto, min-content, max-content)
Columns: count + size (same options)
Row gap: number + unit (px, rem, %)
Column gap: number + unit
Per-track overrides: optional, lets a specific row or column have a different size from the repeat(N, ...) default
Named areas: optional, slot names assigned to cells

For Flex:

Direction: row | column | row-reverse | column-reverse
Wrap: nowrap | wrap | wrap-reverse
Justify content: flex-start | center | space-between | space-around | space-evenly
Align items: stretch | flex-start | center | flex-end | baseline
Gap: number + unit
Per-child overrides: flex-grow, flex-shrink, flex-basis

The CSS generator
rubyclass GridLayoutGenerator
  def initialize(config, layout_id)
    @config = config
    @id = layout_id
  end

  def call
    rules = [container_rule]
    rules << area_rules if @config["areas"]
    rules.compact.join("\n\n")
  end

  private

  def selector = ".layout-#{@id}"

  def container_rule
    props = {
      "display" => "grid",
      "grid-template-rows"    => template(@config["rows"], @config.dig("trackOverrides", "rows")),
      "grid-template-columns" => template(@config["columns"], @config.dig("trackOverrides", "columns")),
      "row-gap"    => unit(@config["rowGap"]),
      "column-gap" => unit(@config["columnGap"])
    }
    props["grid-template-areas"] = format_areas(@config["areas"]) if @config["areas"]

    body = props.map { |k, v| "  #{k}: #{v};" }.join("\n")
    "#{selector} {\n#{body}\n}"
  end

  def template(track, overrides)
    overrides ||= {}
    return "repeat(#{track['count']}, #{unit(track['size'])})" if overrides.empty?

    # if any track is overridden, expand to explicit list
    (0...track["count"]).map { |i| unit(overrides[i.to_s] || track["size"]) }.join(" ")
  end

  def unit(spec)
    return spec if spec.is_a?(String)   # for "auto", "min-content", etc.
    "#{spec['value']}#{spec['unit']}"
  end

  def format_areas(rows)
    rows.map { |row| %("#{row.join(' ')}") }.join("\n    ")
  end

  def area_rules
    slots = @config["areas"].flatten.uniq.reject { |s| s == "." }
    slots.map { |s| "#{selector} > .slot-#{s} { grid-area: #{s}; }" }.join("\n")
  end
end

Why the generator-per-layout approach works here
Brad Woods' tool generates one CSS block for one layout — that's exactly what you need per layout row. The generator runs at save time:
rubyclass Layout < ApplicationRecord
  validates :name, presence: true, uniqueness: true
  validates :type, inclusion: { in: %w[grid flex] }

  before_save :regenerate

  private

  def regenerate
    generator = type == "grid" ? GridLayoutGenerator : FlexLayoutGenerator
    self.compiled_css = generator.new(config, id || SecureRandom.hex(4)).call
    self.compiled_digest = Digest::SHA256.hexdigest(compiled_css)
  end
end
The id || SecureRandom.hex(4) handles the chicken-and-egg of needing an ID before save — alternative is a before_save and after_create two-phase, or just regenerate in after_save once the ID exists. Pick whichever you prefer.

Delivering the CSS
Same pattern as the function URLs — digest-stamped, immutable cache:

class LayoutsController < ApplicationController
  def show
    layout = Layout.find(params[:id])
    response.set_header("Cache-Control", "public, max-age=31536000, immutable") if params[:digest] == layout.compiled_digest
    render plain: layout.compiled_css, content_type: "text/css"
  end
end

Since the slot wrapper has class slot-header (or whatever), a module can mutate that element's classes to add layout behaviour. But — and this is the gap — the slot's grid-area is set by the parent layout's CSS, which the module can't change without touching the page-level stylesheet.
What modules can change at runtime:

Their own internal layout (the inside of the mount point — set display: flex, add padding, etc.)
CSS properties via inline style (this.element.style.gridColumn = "span 2" overrides the layout's positioning)
Adding utility classes from a separate utility stylesheet for things like padding, scroll, etc.

If you want modules to be able to declare "I want to span 2 columns instead of 1," the cleanest way is to let them set style="grid-column: span 2" directly. That overrides the layout's grid-area rule via specificity (inline styles win).
For the "modules modify slot classes" idea from earlier: keep that as a separate utility-class layer for behaviours like slot-scroll, slot-pad, etc. — orthogonal to the layout-grid-positioning that comes from the generated CSS. Two layers:

Generated layout CSS (this turn's topic) — positions slots in the grid via grid-area/grid-column/grid-row.
Static utility CSS — a hand-written file with .slot-scroll, .slot-pad, etc., that modules add/remove via classList.

They don't conflict because they target different properties.

Validation. Named areas in CSS Grid have to form rectangles, per spec. The browser silently rejects the entire grid-template-areas declaration if any area is non-rectangular. Validate this in the model:
rubydef areas_are_rectangular
  return unless config["areas"]
  config["areas"].flatten.uniq.reject { |s| s == "." }.each do |slot|
    cells = []
    config["areas"].each_with_index do |row, r|
      row.each_with_index { |cell, c| cells << [r, c] if cell == slot }
    end
    rows = cells.map(&:first).uniq.sort
    cols = cells.map(&:last).uniq.sort
    expected = rows.product(cols).sort
    errors.add(:config, "slot '#{slot}' is not rectangular") unless cells.sort == expected
  end
end

The root controller
javascript// app/javascript/controllers/layout_editor_controller.js
import { Controller } from "@hotwired/stimulus"

export default class extends Controller {
  static values = {
    config: Object,
    id: Number,
    saveUrl: String
  }

  static targets = ["cssOutput", "previewMount"]

  connect() {
    this.config = structuredClone(this.configValue)
    this.regenerate()
  }

  updateConfig(event) {
    const { path, value } = event.detail
    this.setPath(this.config, path, value)
    this.regenerate()
  }

  updateAreas(event) {
    this.config.areas = event.detail.areas
    this.regenerate()
  }

  regenerate() {
    const css = generateCss(this.config, this.idValue)
    this.cssOutputTarget.textContent = css
    this.renderPreview(css)
  }

  renderPreview(css) {
    // wipe and rebuild
    this.previewMountTarget.innerHTML = ""

    const style = document.createElement("style")
    style.textContent = css
    this.previewMountTarget.appendChild(style)

    const layout = document.createElement("section")
    layout.className = `layout-${this.idValue}`

    const slotNames = (this.config.areas || []).flat().filter((s, i, a) => s !== "." && a.indexOf(s) === i)
    const count = slotNames.length || this.config.childrenCount || 9

    if (slotNames.length) {
      slotNames.forEach((name, i) => {
        const cell = document.createElement("div")
        cell.className = `slot-${name}`
        cell.textContent = name
        layout.appendChild(cell)
      })
    } else {
      for (let i = 1; i <= count; i++) {
        const cell = document.createElement("div")
        cell.textContent = i
        layout.appendChild(cell)
      }
    }

    this.previewMountTarget.appendChild(layout)
  }

  async save() {
    const response = await fetch(this.saveUrlValue, {
      method: "PATCH",
      headers: {
        "Content-Type": "application/json",
        "X-CSRF-Token": document.querySelector("meta[name='csrf-token']").content,
        "Accept": "application/json"
      },
      body: JSON.stringify({ layout: { config: this.config } })
    })
    if (!response.ok) {
      const errors = await response.json()
      this.dispatch("save-error", { detail: errors })
    } else {
      this.dispatch("saved")
    }
  }

  setPath(obj, path, value) {
    const keys = path.split(".")
    const last = keys.pop()
    const target = keys.reduce((o, k) => (o[k] ??= {}), obj)
    target[last] = value
  }
}

javascript// app/javascript/controllers/area_grid_controller.js
import { Controller } from "@hotwired/stimulus"

export default class extends Controller {
  static values = { rows: Number, columns: Number, areas: Array }

  connect() {
    this.areas = this.areasValue.length
      ? this.areasValue
      : this.emptyGrid(this.rowsValue, this.columnsValue)
    this.render()
  }

  rowsValueChanged()    { this.resize() }
  columnsValueChanged() { this.resize() }

  resize() {
    const next = this.emptyGrid(this.rowsValue, this.columnsValue)
    for (let r = 0; r < Math.min(this.areas.length, this.rowsValue); r++) {
      for (let c = 0; c < Math.min(this.areas[r].length, this.columnsValue); c++) {
        next[r][c] = this.areas[r][c]
      }
    }
    this.areas = next
    this.render()
    this.emit()
  }

  emptyGrid(r, c) {
    return Array.from({ length: r }, () => Array.from({ length: c }, () => "."))
  }

  render() {
    this.element.innerHTML = ""
    this.element.style.display = "grid"
    this.element.style.gridTemplateColumns = `repeat(${this.columnsValue}, 1fr)`
    this.element.style.gap = "4px"

    this.areas.forEach((row, r) => {
      row.forEach((cell, c) => {
        const input = document.createElement("input")
        input.type = "text"
        input.value = cell === "." ? "" : cell
        input.placeholder = "+ name"
        input.dataset.row = r
        input.dataset.col = c
        input.addEventListener("input", e => this.cellChanged(e))
        this.element.appendChild(input)
      })
    })
  }

  cellChanged(event) {
    const r = Number(event.target.dataset.row)
    const c = Number(event.target.dataset.col)
    this.areas[r][c] = event.target.value || "."
    this.emit()
  }

  emit() {
    this.dispatch("change", {
      detail: { areas: this.areas },
      bubbles: true
    })
  }
}

javascriptfunction validateAreas(areas) {
  const errors = []
  const slots = [...new Set(areas.flat())].filter(s => s !== ".")
  for (const slot of slots) {
    const cells = []
    areas.forEach((row, r) => row.forEach((c, ci) => c === slot && cells.push([r, ci])))
    const rows = [...new Set(cells.map(c => c[0]))]
    const cols = [...new Set(cells.map(c => c[1]))]
    if (cells.length !== rows.length * cols.length) {
      errors.push(`'${slot}' must be rectangular`)
    }
  }
  return errors
}

The CSS generator port. Maintaining two implementations (Ruby + JS) is annoying. Two alternatives:

Make the editor save on every keystroke (debounced, 300ms) and re-fetch the server-rendered CSS. Simpler code, more requests. For an internal-use editor, fine.

structuredClone. MDN structuredClone is available in all modern browsers since 2022. If you need older support, JSON.parse(JSON.stringify(x)) is the fallback for plain JSON-shaped objects.
textContent for CSS injection in preview. Using <style> with textContent is safe (no HTML parsing). Avoids any XSS concerns from user-typed slot names.
Debouncing. The preview regenerates on every keystroke. For typical input volumes that's fine, but if regenerate ever becomes expensive (e.g. you add an iframe-based preview), debounce in updateConfig with a 100ms timer.
Save button vs autosave. The above is explicit-save. For an experimental editor, autosave on blur or after debounced inactivity is friendlier — fewer keystrokes lost. Adds complexity around "is this edit still pending" state. I'd start with explicit save and add autosave once the editor is in real use.
